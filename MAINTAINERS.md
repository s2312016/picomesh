## Contact

- Usage questions and reproducible bugs: GitHub Issues
- Proposed API or protocol changes: GitHub Issues followed by a pull request
- Security vulnerabilities: GitHub private vulnerability reporting, as described in `SECURITY.md`

## Becoming a core maintainer

Core-maintainer access may be offered after sustained contributions that include code review, issue triage, documentation, testing, or release work. Maintainers must follow `GOVERNANCE.md`, preserve deterministic resource bounds, and avoid introducing private or incompatibly licensed material.
